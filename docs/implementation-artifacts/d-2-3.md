## Story

As a 系统管理员,
I want 将现有内存存储数据迁移到MySQL数据库,
so that 保障数据持久化，支持应用重启后数据恢复。

根据 PRD v2 - MySQL数据库迁移 (docs/planning-artifacts/prd-v2-mysql.md)，数据迁移是 MySQL 迁移的关键步骤。

**当前进度：**

- d-1-2: Configuration Management ✅ 已完成

**迁移策略（PRD 2.3）：**
- 方案：一次性迁移
- 停止服务 → 导出内存数据 → 初始化MySQL表结构 → 导入数据 → 验证数据完整性 → 启动服务

### 项目结构

**相关文件：**
- 数据库配置: `src/config/database.config.ts`
- Prisma Client: `src/database/prisma-client.ts`
- Prisma Schema: `prisma/schema.prisma`
- MySQL Storage Adapter: `src/repositories/mysql-storage.adapter.ts`
- Repository Factory: `src/repositories/repository.factory.ts`

**环境变量配置：**
```bash
# .env 文件配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=cregis_openplatform
DB_USER=root
DB_PASSWORD=your_password
STORAGE_DRIVER=mysql
STORAGE_TYPE=mysql
DATABASE_URL=mysql://root:password@localhost:3306/cregis_openplatform
```

### 架构约束

- **ORM**: Prisma (类型安全，支持自动迁移)
- **迁移工具**: Prisma Migrate
- **连接池**: Prisma 内置（默认10连接）
- **数据库**: MySQL 8.0+

### 测试标准

- 数据库连接测试通过
- CRUD 操作测试通过
- 数据完整性验证

### 注意事项

1. 确保 MySQL 8.0+ 已安装并运行
2. 创建对应的数据库: `CREATE DATABASE cregis_openplatform`
3. 当前 Repository 层已支持 MySQL，切换存储驱动即可

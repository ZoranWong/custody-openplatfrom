## Dev Notes

### Technical Background

**签名算法 (必须与后端一致):**

```
1. 对business JSON按key排序后序列化: JSON.stringify(sortKeys(business))
2. 计算business的MD5: md5 = MD5(序列化后的JSON)
3. 拼接签名字符串: appId + resourceKey + timestamp + nonce + md5(business)
4. 使用appSecret签名: MD5(appSecret + 签名字符串)
5. 将签名结果放入basic.signature字段
```

**请求格式:**

```json
{
  "basic": {
    "appId": "uuid",
    "resourceKey": "string",
    "timestamp": 1234567890,
    "nonce": "random-string",
    "signature": "md5-hash"
  },
  "business": {
    // 业务数据
  }
}
```

### Architecture Constraints

- **语言**: TypeScript (Node.js SDK)
- **加密算法**: MD5 (必须与后端一致)
- **序列化**: JSON.stringify with sorted keys
- **签名格式**: MD5(appSecret + appId + resourceKey + timestamp + nonce + md5(business))

### Implementation思路

1. 创建 `openplatform-sdk/nodejs/src/utils/signature.ts`
2. 实现 `generateNonce()` - 生成随机字符串
3. 实现 `sortObjectKeys()` - 递归排序对象key
4. 实现 `calculateBusinessMd5()` - 计算business数据的MD5
5. 实现 `buildSignatureString()` - 拼接签名字符串 (包含resourceKey)
6. 实现 `generateSignature()` - 生成签名
7. 实现 `buildSignedRequest()` - 构建完整签名请求

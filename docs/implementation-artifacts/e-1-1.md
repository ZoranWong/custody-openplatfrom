# Story e-1-1: Resource Validation Middleware (Developer Identity + Signature Verification)

Status: done

## Story

As a **API Gateway**,
I want to verify the developer's identity and signature from request basic info,
so that I can ensure the request is from an authenticated and authorized developer with valid signature.

## Acceptance Criteria

1. [AC-1] Extract appId from request body.basic
2. [AC-2] Verify all required basic fields exist (appId, resourceKey, timestamp, nonce, signature)
3. [AC-3] Validate appId format (UUID)
4. [AC-4] Verify appId exists and is active
5. [AC-5] Retrieve app associated developer information
6. [AC-6] Verify developer account is active
7. [AC-7] Validate timestamp is within acceptable range (5 minutes)
8. [AC-8] Check nonce for replay attack prevention
9. [AC-9] Verify signature using MD5(appSecret + signString)
10. [AC-10] Attach developer info and resource info to request context
11. [AC-11] Return 400 if basic info missing/incomplete
12. [AC-12] Return 401 if appId invalid, not active, or signature invalid

## Dev Notes

### Technical Background

根据 Epic E (Custody 消息转发签名验证) 的设计，开发者身份验证和签名验证基于请求body中的basic信息。

**请求格式:**

```json
{
  "basic": {
    "appId": "uuid",
    "resourceKey": "string",
    "timestamp": number,
    "nonce": "string",
    "signature": "string"
  },
  "business": {
    // 业务数据
  }
}
```

### 签名算法

```
1. 对business JSON按key排序后序列化: JSON.stringify(sortKeys(business))
2. 计算business的MD5: md5 = MD5(序列化后的JSON)
3. 拼接签名字符串: appId + timestamp + nonce + md5
4. 使用appSecret签名: MD5(appSecret + 签名字符串)
5. 将签名结果放入basic.signature字段
```

### 项目结构

**文件位置:**
- 中间件: `src/middleware/resource-validation.middleware.ts`

### 架构约束

- **身份来源**: 请求body.basic中的appId
- **验证方式**: 查询Application表获取关联开发者信息 + MD5签名验证
- **请求方法**: 仅支持POST
- **数据格式**: 严格按{ basic: {...}, business: {...} }格式

### 实现思路

1. 创建ResourceValidationMiddleware类
2. 从请求body.basic提取appId, resourceKey, timestamp, nonce, signature
3. 验证所有basic字段必须存在
4. 查询Application表验证状态并获取开发者信息
5. 验证timestamp在允许范围内
6. 检查nonce防止重放攻击
7. 使用MD5验证签名
8. 将开发者/App信息附加到req.developerContext
9. 附加resourceKey和businessData到请求对象
10. 错误处理返回标准400/401响应

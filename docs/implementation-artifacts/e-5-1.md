## Dev Notes

### Technical Background

授权转发通过 Forwarder 机制实现，支持可扩展的转发策略。

**架构设计:**
- **Forwarder 接口**: 定义统一的转发接口
- **DefaultForwarder**: 标准 HTTP 转发
- **CustodyForwarder**: Custody 消息转发，带签名验证
- **ForwarderRegistry**: 转发器注册表

**关键特性:**
- 路由配置支持 `forwarder` 字段指定转发器
- 路由配置支持 `forwardRoutes` 配置每个路径的 HTTP 方法
- 统一入口：客户端所有请求使用 POST，通过配置决定实际转发方法

**请求流程:**
```
Client POST /api/v1/custody/enterprise/wallets
    ↓
RequestRoutingService.findRouteWithInfo() (一次查询)
    ↓
CustodyForwarder.forward(route, req)
    ↓ (根据 forwardRoutes 配置)
Backend Service (GET /api/v1/enterprise/wallets)
```

### Implementation Note

- 删除了独立的 custody-forwarding.middleware
- 改用 Forwarder 机制，集成到 request-routing 中
- 优化了路由查询，避免重复计算

### Architecture Compliance

- 遵循现有中间件模式
- 使用统一的错误响应格式
- 集成现有日志系统
- 支持配置化的服务 endpoint

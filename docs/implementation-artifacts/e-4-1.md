# Story e-4-1: Resource Authorization Service

Status: done

## Story

As a **Authorization Service**,
I want to verify that the appId has permission to access the resourceKey,
so that I can enforce access control.

## Acceptance Criteria

1. [AC-1] Query authorization records by appId and resourceKey
2. [AC-2] Check if authorization is active and not expired
3. [AC-3] Verify authorization scope covers the requested operation
4. [AC-4] Return authorization result with details
5. [AC-5] Return 403 if not authorized

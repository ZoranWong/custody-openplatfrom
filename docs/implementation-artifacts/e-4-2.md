## Story

As a **System**,
I want to cache authorization results,
so that I can improve performance for repeated requests.

## Dev Notes

### Technical Background

授权缓存层用于缓存授权验证结果，提高重复请求的性能。

**注意:** 此Story标记为Optional，当前实现中已有简单的in-memory nonce cache。

**实现方案:**
- 使用内存缓存 (Map)
- TTL-based过期机制
- 支持缓存失效

### Architecture

**文件位置:**
- 缓存: `src/services/authorization-cache.service.ts`
- 服务: `src/services/resource-authorization.service.ts` (更新)

### Implementation Note

此功能为可选优化，当前授权验证性能已足够。Redis缓存可在后续需要分布式缓存时添加。

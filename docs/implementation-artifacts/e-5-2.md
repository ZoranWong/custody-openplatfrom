## Dev Notes

### Technical Background

Epic E 的错误处理和日志需要规范化。目前存在的问题：
1. `resource-validation.middleware.ts` 使用 `console.error` 直接输出
2. `forwarders.ts` 使用简单 console.logger
3. 需要统一使用结构化日志系统

**现有日志系统:**
- `src/utils/logger.ts` - Winston 包装器，支持结构化 JSON 日志
- `src/services/error-mapper.service.ts` - 错误码映射服务

**错误码规范:**
- 40001-40099: 参数错误
- 40101-40199: 认证错误
- 40301-40399: 授权错误
- 50001-50099: 服务器错误

### Implementation Approach

**日志脱敏规则:**
- `signature`: 显示前4位和后4位，中间用 `***` 替换
- `appSecret`: 不记录日志
- `password`: 不记录日志
- `token`: 显示前8位

**签名验证失败日志:**
```json
{
  "type": "audit",
  "event": "signature_verification_failed",
  "appId": "app_123",
  "resourceKey": "enterprise:wallets",
  "timestamp": 1706745600,
  "nonce": "abc123",
  "signature": "abc1***7890",
  "reason": "signature_mismatch",
  "trace_id": "req_xxx"
}
```

**授权失败日志:**
```json
{
  "type": "audit",
  "event": "authorization_failed",
  "appId": "app_123",
  "resourceKey": "enterprise:wallets",
  "reason": "authorization_expired",
  "expiresAt": "2026-01-01T00:00:00Z",
  "trace_id": "req_xxx"
}
```

### Architecture Compliance

- 遵循现有 Winston logger 模式
- 错误响应使用统一的 errorMapper
- 敏感信息脱敏符合安全要求
- 审计日志使用 `type: audit` 标记
